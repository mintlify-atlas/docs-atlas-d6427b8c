---
title: Knowledge Graph Types
description: TypeScript interfaces and types that define the structure of the knowledge graph produced by Understand Anything.
---

All types are exported from `@understand-anything/core` (main entry) and from the `@understand-anything/core/types` subpath export (browser-safe).

```typescript
import type { KnowledgeGraph, GraphNode, GraphEdge, EdgeType } from '@understand-anything/core/types';
```

---

## KnowledgeGraph

The root structure produced by the analysis pipeline and stored in `.understand-anything/knowledge-graph.json`.

```typescript
export interface KnowledgeGraph {
  version: string;
  project: ProjectMeta;
  nodes: GraphNode[];
  edges: GraphEdge[];
  layers: Layer[];
  tour: TourStep[];
}
```

<ResponseField name="version" type="string" required>
  Schema version string (e.g. `"1.0.0"`). Used for forward-compatibility checks.
</ResponseField>

<ResponseField name="project" type="ProjectMeta" required>
  Metadata about the analyzed project.

  <Expandable title="ProjectMeta fields">
    <ResponseField name="name" type="string" required>
      Display name of the project.
    </ResponseField>
    <ResponseField name="languages" type="string[]" required>
      Sorted list of detected programming languages (e.g. `["javascript", "typescript"]`).
    </ResponseField>
    <ResponseField name="frameworks" type="string[]" required>
      Detected frameworks. May be empty when auto-detection is not available.
    </ResponseField>
    <ResponseField name="description" type="string" required>
      Human-readable project description generated by the LLM.
    </ResponseField>
    <ResponseField name="analyzedAt" type="string" required>
      ISO 8601 timestamp of when the analysis was run.
    </ResponseField>
    <ResponseField name="gitCommitHash" type="string" required>
      Git commit SHA at the time of analysis. Used for staleness checks.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="nodes" type="GraphNode[]" required>
  All nodes in the graph. See [GraphNode](#graphnode) below.
</ResponseField>

<ResponseField name="edges" type="GraphEdge[]" required>
  All directed edges between nodes. See [GraphEdge](#graphedge) below.
</ResponseField>

<ResponseField name="layers" type="Layer[]" required>
  Logical groupings of nodes (e.g. "API Layer", "Data Layer"). May be empty.
</ResponseField>

<ResponseField name="tour" type="TourStep[]" required>
  Ordered steps for the guided learning tour. May be empty.
</ResponseField>

## GraphNode

Represents a single entity in the codebase — a file, function, class, module, or abstract concept.

```typescript
export interface GraphNode {
  id: string;
  type: "file" | "function" | "class" | "module" | "concept";
  name: string;
  filePath?: string;
  lineRange?: [number, number];
  summary: string;
  tags: string[];
  complexity: "simple" | "moderate" | "complex";
  languageNotes?: string;
}
```

<ResponseField name="id" type="string" required>
  Stable unique identifier. Follows a namespaced convention:
  - `file:<path>` — e.g. `file:src/index.ts`
  - `func:<path>:<name>` — e.g. `func:src/auth.ts:validateToken`
  - `class:<path>:<name>` — e.g. `class:src/server.ts:HttpServer`
</ResponseField>

<ResponseField name="type" type='"file" | "function" | "class" | "module" | "concept"' required>
  The kind of entity this node represents.
</ResponseField>

<ResponseField name="name" type="string" required>
  Short display name. For file nodes this is the filename; for functions and classes it is the symbol name.
</ResponseField>

<ResponseField name="filePath" type="string">
  Relative path to the source file. Present on `file`, `function`, and `class` nodes. Omitted for `module` and `concept` nodes.
</ResponseField>

<ResponseField name="lineRange" type="[number, number]">
  Inclusive start/end line numbers within `filePath`. Only present on `function` and `class` nodes.
</ResponseField>

<ResponseField name="summary" type="string" required>
  LLM-generated plain-English description of what this entity does.
</ResponseField>

<ResponseField name="tags" type="string[]" required>
  Keyword tags used for search ranking. May be empty for auto-generated nodes.
</ResponseField>

<ResponseField name="complexity" type='"simple" | "moderate" | "complex"' required>
  Estimated complexity level of the entity.
</ResponseField>

<ResponseField name="languageNotes" type="string">
  Optional notes about language-specific idioms or patterns. Contributes to fuzzy search with weight `0.1`.
</ResponseField>

## GraphEdge

A directed relationship between two nodes.

```typescript
export interface GraphEdge {
  source: string;
  target: string;
  type: EdgeType;
  direction: "forward" | "backward" | "bidirectional";
  description?: string;
  weight: number; // 0–1
}
```

<ResponseField name="source" type="string" required>
  `id` of the originating node.
</ResponseField>

<ResponseField name="target" type="string" required>
  `id` of the destination node.
</ResponseField>

<ResponseField name="type" type="EdgeType" required>
  The semantic relationship type. See [EdgeType](#edgetype) for all 18 values.
</ResponseField>

<ResponseField name="direction" type='"forward" | "backward" | "bidirectional"' required>
  Directionality of the relationship relative to `source → target`.
</ResponseField>

<ResponseField name="description" type="string">
  Optional human-readable annotation for the edge.
</ResponseField>

<ResponseField name="weight" type="number" required>
  Strength of the relationship in the range `0–1`. Used by graph layout algorithms. Default weights assigned by `GraphBuilder`:
  - `imports` edges: `0.7`
  - `calls` edges: `0.8`
  - `contains` edges: `1.0`
</ResponseField>

## EdgeType

Union of all 18 possible edge relationship types, organized into 5 semantic categories.

```typescript
export type EdgeType =
  | "imports" | "exports" | "contains" | "inherits" | "implements"  // Structural
  | "calls" | "subscribes" | "publishes" | "middleware"              // Behavioral
  | "reads_from" | "writes_to" | "transforms" | "validates"         // Data flow
  | "depends_on" | "tested_by" | "configures"                       // Dependencies
  | "related" | "similar_to";                                        // Semantic
```

**Structural** — static code structure relationships:

| Value | Meaning |
|---|---|
| `imports` | Source file imports the target |
| `exports` | Source exposes the target as a public API |
| `contains` | Source structurally contains the target (e.g. file → function) |
| `inherits` | Source class extends the target class |
| `implements` | Source class implements the target interface |

**Behavioral** — runtime interaction relationships:

| Value | Meaning |
|---|---|
| `calls` | Source function invokes the target function |
| `subscribes` | Source subscribes to events or messages from the target |
| `publishes` | Source emits events or messages consumed by the target |
| `middleware` | Source is a middleware layer that wraps the target |

**Data flow** — data transformation relationships:

| Value | Meaning |
|---|---|
| `reads_from` | Source reads data produced by the target |
| `writes_to` | Source writes data consumed by the target |
| `transforms` | Source transforms the output of the target |
| `validates` | Source validates the schema or output of the target |

**Dependencies** — build and test relationships:

| Value | Meaning |
|---|---|
| `depends_on` | Source requires the target to function |
| `tested_by` | Source is tested by the target |
| `configures` | Source configures or initializes the target |

**Semantic** — loose conceptual relationships:

| Value | Meaning |
|---|---|
| `related` | Source and target are conceptually related |
| `similar_to` | Source and target serve a similar purpose |

## Layer

A logical grouping of nodes used to represent architectural layers in the dashboard.

```typescript
export interface Layer {
  id: string;
  name: string;
  description: string;
  nodeIds: string[];
}
```

<ResponseField name="id" type="string" required>
  Stable unique identifier for the layer.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name (e.g. `"API Layer"`, `"Data Access Layer"`).
</ResponseField>

<ResponseField name="description" type="string" required>
  LLM-generated description of the layer's responsibility.
</ResponseField>

<ResponseField name="nodeIds" type="string[]" required>
  Array of `GraphNode.id` values belonging to this layer.
</ResponseField>

## TourStep

One step in the guided onboarding tour, used by the Learn persona in the dashboard.

```typescript
export interface TourStep {
  order: number;
  title: string;
  description: string;
  nodeIds: string[];
  languageLesson?: string;
}
```

<ResponseField name="order" type="number" required>
  1-based position of this step in the tour sequence.
</ResponseField>

<ResponseField name="title" type="string" required>
  Short title displayed in the learn panel.
</ResponseField>

<ResponseField name="description" type="string" required>
  Explanation of what the user should understand at this step.
</ResponseField>

<ResponseField name="nodeIds" type="string[]" required>
  Nodes highlighted in the graph when this step is active.
</ResponseField>

<ResponseField name="languageLesson" type="string">
  Optional language-specific lesson content (Markdown). Present when the step focuses on a language idiom.
</ResponseField>

## AnalysisMeta

Persistence metadata written alongside the graph. Not part of `KnowledgeGraph` itself.

```typescript
export interface AnalysisMeta {
  lastAnalyzedAt: string;
  gitCommitHash: string;
  version: string;
  analyzedFiles: number;
}
```

<ResponseField name="lastAnalyzedAt" type="string" required>
  ISO 8601 timestamp of the most recent analysis.
</ResponseField>

<ResponseField name="gitCommitHash" type="string" required>
  Git commit SHA from the most recent analysis run.
</ResponseField>

<ResponseField name="version" type="string" required>
  Core package version used to produce the stored graph.
</ResponseField>

<ResponseField name="analyzedFiles" type="number" required>
  Number of files included in the analysis.
</ResponseField>

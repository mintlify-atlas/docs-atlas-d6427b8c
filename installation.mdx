# Installation

Understand Anything works across all major AI coding platforms. Choose your platform below.

## Platform compatibility

| Platform | Status | Install method |
|----------|--------|----------------|
| Claude Code | Native | Plugin marketplace |
| Codex | Supported | AI-driven install |
| OpenCode | Supported | Plugin config (`opencode.json`) |
| OpenClaw | Supported | AI-driven install |
| Cursor | Supported | Auto-discovery (clone repo) |
| Antigravity | Supported | AI-driven install |

<Tabs>
  <Tab title="Claude Code">
    Claude Code has native plugin support. Install directly from the marketplace:

    <Steps>
      <Step title="Add from marketplace">
        ```bash
        /plugin marketplace add Lum1104/Understand-Anything
        ```
      </Step>
      <Step title="Install the plugin">
        ```bash
        /plugin install understand-anything
        ```
      </Step>
      <Step title="Verify installation">
        Run the analysis command in any project:

        ```bash
        /understand
        ```

        The multi-agent pipeline starts immediately.
      </Step>
    </Steps>

    <Tip>
      This is the native, fully supported platform. All six commands (`/understand`, `/understand-dashboard`, `/understand-chat`, `/understand-diff`, `/understand-explain`, `/understand-onboard`) are available out of the box.
    </Tip>
  </Tab>

  <Tab title="OpenCode">
    OpenCode uses a `opencode.json` plugin config file. The plugin auto-installs and registers all skills on startup.

    <Steps>
      <Step title="Add to opencode.json">
        Add the plugin entry to your `opencode.json` (global or project-level):

        ```json
        {
          "plugin": ["understand-anything@git+https://github.com/Lum1104/Understand-Anything.git"]
        }
        ```
      </Step>
      <Step title="Restart OpenCode">
        Restart OpenCode. The plugin is automatically fetched from GitHub and all skills are registered.
      </Step>
      <Step title="Verify installation">
        Ask OpenCode to list available skills. You should see `understand`, `understand-chat`, `understand-dashboard`, and more.

        Then load the skill to analyze your codebase:

        ```
        use skill tool to load understand-anything/understand
        ```

        Or simply ask: `"Analyze this codebase and build a knowledge graph"`
      </Step>
    </Steps>

    **Pinning a specific version:**

    ```json
    {
      "plugin": ["understand-anything@git+https://github.com/Lum1104/Understand-Anything.git#v1.1.1"]
    }
    ```

    **Updating:** Restart OpenCode â€” the plugin re-installs from git automatically.

    **Uninstalling:** Remove the plugin line from `opencode.json` and restart.

    <Note>
      When OpenCode skills reference Claude Code-specific tools, the mapping is: `TodoWrite` â†’ `todowrite`, `Task` with subagents â†’ `@mention` syntax, `Skill` tool â†’ OpenCode's native `skill` tool.
    </Note>
  </Tab>

## Troubleshooting

### Skills not found after install (OpenCode)

1. Verify the plugin line exists in your `opencode.json`
2. Check that `~/.cache/opencode/node_modules/understand-anything` was created after restart
3. Use the `skill` tool to list discovered skills

### Analysis stops at >200 files

For very large projects, the pipeline prompts for confirmation before proceeding. You can also scope the analysis to a subdirectory:

```bash
/understand src/
```

### Force a full rebuild

If the graph appears stale or incomplete, force a full rebuild:

```bash
/understand --full
```
